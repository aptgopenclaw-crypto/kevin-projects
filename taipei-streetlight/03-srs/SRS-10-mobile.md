## SRS-10-004 拍照管控

**來源**：§10-(4)

### User Story

> 身為**系統**，需確保維護/巡查照片的真實性：僅限資產周邊 50 公尺內拍攝，強制加入位置與時間浮水印，非 APP 拍攝照片需 EXIF 驗證。

### 主要流程

**APP 內拍照**
1. 開啟相機前驗證 GPS 定位是否在資產 50m 範圍內
2. 超出範圍時拒絕拍照並提示
3. 拍攝時強制開啟位置標籤
4. 照片嵌入日期/時間浮水印（不可移除）
5. 上傳時自動記錄 GPS、時間至 ticket_attachments

**非 APP 照片匯入**
1. 使用者上傳外部照片
2. 系統解析 EXIF 資訊（GPS、時間、相機型號）
3. 驗證為原始未修改版本（完整性檢查）
4. 比對 EXIF 位置/時間與資產/案件是否吻合
5. 不符合時標記異常，需機關同意後始得匯入

### 驗收準則

| AC ID | 驗收條件 |
|-------|---------|
| AC-10-004-1 | APP 拍照前驗證 GPS 距離 ≤ 50m（可設定） |
| AC-10-004-2 | 超出範圍拒絕拍照並提示距離 |
| AC-10-004-3 | 可設定強制開啟位置標籤 |
| AC-10-004-4 | 照片自動嵌入日期/時間浮水印（防篡改） |
| AC-10-004-5 | 非 APP 照片可解析 EXIF 並驗證原始性 |
| AC-10-004-6 | EXIF 位置/時間不符時標記異常，需機關同意才可異動 |

### 技術設計

- GPS 距離計算：Haversine 公式
- EXIF 解析：ExifInterface (Android) / CGImageSource (iOS) / 後端 Apache Commons Imaging
- 浮水印：拍攝時 Canvas 繪製（不可事後移除）
- 完整性：比對 EXIF 原始資料 hash

## SRS-10-005 離線續傳

**來源**：§10-(5)

### User Story

> 身為 **CTR_FIELD**，現場無網路時 APP 仍可操作記錄資料，恢復網路後自動續傳。

### 主要流程

1. APP 偵測到網路中斷
2. 所有操作暫存至本地 SQLite 資料庫
3. 照片暫存於裝置本地空間
4. 恢復網路後自動觸發同步
5. 照片採分塊上傳（chunked upload），斷點續傳
6. 同步完成後通知使用者

### 驗收準則

| AC ID | 驗收條件 |
|-------|---------|
| AC-10-005-1 | 無網路時可繼續操作（普查/巡查/維護記錄/拍照） |
| AC-10-005-2 | 恢復網路後自動同步（不需使用者手動觸發） |
| AC-10-005-3 | 照片支援斷點續傳 |
| AC-10-005-4 | 同步衝突時以伺服器端為主，提示使用者 |
| AC-10-005-5 | 同步狀態可視化（進度/成功/失敗） |

### 技術設計

- **離線儲存**：SQLite（結構化資料）+ 本地檔案系統（照片/影片）
- **同步策略**：Queue-based sync manager，優先級排序（先同步文字再同步媒體）
- **分塊上傳**：5MB chunks，每 chunk 獨立確認，失敗重試
- **衝突解決**：Last-write-wins + 伺服器版本號比對
