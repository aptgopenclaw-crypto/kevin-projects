> **文件版本**：1.0  
> **發行日期**：2026-04-24  
> **需求來源**：〈臺北市路燈管理平台需求規格書〉（甲方 RFP 附件）  
> **設計參照**：`/02-spec/` 系統設計規格書、`/_archive/x-plan/` 分階段實施計畫

---

## 1. 文件目的

本 SRS 將甲方需求規格書 §1–§11 及效能需求，逐條展開為**可追溯、可驗收**的系統需求項目。每條 SRS 包含：

- **需求追溯**：對應原始需求編號（§ 章節）
- **Use Case / User Story**：角色、前置條件、主要流程、例外處理、後置條件
- **驗收準則**（Acceptance Criteria）：明確、可測試的通過條件
- **資料模型概要**：核心 Entity 與關聯
- **API 端點概要**：RESTful 路徑與 HTTP Method
- **實作狀態**：已完成 ✅ / 部分完成 ⚠️ / 未開始 ❌

## 3. 需求編號規則

```
SRS-MM-NNN
 │    │  └── 流水號（001–999）
 │    └───── 模組編號（01–11, NFR）
 └────────── 固定前綴
```

範例：`SRS-02-003` = 系統管理模組第 3 條需求（登入管理）

|------|---------|------|
| 前端 Web | Vue 3 + TypeScript + Element Plus | i18n: zh-TW / en / zh-CN |
| 前端 APP | Flutter / React Native（待定） | 共用後端 API |
| 後端 API | Spring Boot 3 + JPA/Hibernate | RESTful，JWT 認證 |
| 資料庫 | PostgreSQL + Flyway | Multi-tenant，JSONB 動態欄位 |
| GIS | ArcGIS Server + OpenLayers | 精確圖資 + 開放瀏覽 |
| IoT | MQTT Broker + TimescaleDB | 時序資料庫 |
| 即時推送 | WebSocket / SSE | 告警、通知 |
| 圖表 | Apache ECharts | 儀表板視覺化 |
| 快取 | Redis | Session、統計快取 |
