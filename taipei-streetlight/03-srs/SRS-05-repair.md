## SRS-05-001 民眾線上報修

**來源**：§5-(1)

### User Story

> 身為 **PUBLIC**（一般民眾），我可透過公開網頁或掃描路燈號碼牌 QR Code 報修，並查詢報修進度。

### 主要流程

1. 民眾透過公開 Web 網頁或掃描燈桿 QR Code 進入報修頁
2. 填寫：姓名、聯絡方式、報修地點、報修項目、上傳照片（限件數/容量）
3. 勾選個資使用聲明同意
4. 提交後取得案件編號，可憑此查詢進度
5. 維修完成後通知民眾

### 驗收準則

| AC ID | 驗收條件 |
|-------|---------|
| AC-05-001-1 | 提供公開報修網頁（不需登入） |
| AC-05-001-2 | 可掃描燈桿 QR Code 自動帶入路燈資訊 |
| AC-05-001-3 | 報修頁面有個資使用聲明，需勾選同意 |
| AC-05-001-4 | 可上傳照片/影片/錄音（限定件數與容量，系統可設定） |
| AC-05-001-5 | 提交後回傳案件編號，可查詢處理進度 |
| AC-05-001-6 | 比照臺北市陳情系統實名制模式 |

### 狀態：❌ 未開始

## SRS-05-004 派工及回報維修處理

**來源**：§5-(4)

### User Story

> 身為 **GOV_STAFF**，我可將報修案件派工給廠商；身為 **CTR_FIELD**，我可線上回報維修處理情形（含照片）。

### 主要流程

1. 承辦人收到報修案件，建立派工單指派至維護廠商
2. 廠商現場人員接收派工，開始維修
3. 維修前/中/後拍照（須記錄日期、時間、GPS）
4. 上傳維修報告，填寫故障原因、更換設備、完工時間
5. 提交完工送審

### 驗收準則

| AC ID | 驗收條件 |
|-------|---------|
| AC-05-004-1 | 可建立派工單，指派至維護廠商 |
| AC-05-004-2 | 維護人員可線上回報維修處理（含前/中/後照片） |
| AC-05-004-3 | 照片須記錄日期、時間、GPS 位置 |
| AC-05-004-4 | 可匯出案件清冊 |

### 資料模型

```
repair_tickets: id, ticket_number, source(CITIZEN_REPORT/PATROL/AUTO_ALERT/EXTERNAL),
                status, device_id, reporter_name, reporter_phone, location_desc,
                fault_category, priority, deadline_at, ...
repair_dispatches: id, ticket_id, contractor_id, assigned_by, dispatched_at,
                   accepted_at, completed_at
ticket_attachments: id, ticket_type, ticket_id, file_path, file_type,
                    phase(BEFORE/DURING/AFTER), gps_lat, gps_lng, taken_at, scan_status
```

### API 端點

| Method | Path | 說明 |
|--------|------|------|
| GET | `/v1/repair/tickets` | 案件列表 |
| POST | `/v1/repair/tickets` | 立案 |
| GET | `/v1/repair/tickets/{id}` | 詳情 |
| POST | `/v1/repair/tickets/{id}/dispatch` | 派工 |
| POST | `/v1/repair/tickets/{id}/accept` | 廠商接收 |
| POST | `/v1/repair/tickets/{id}/report-completion` | 完工回報 |
| POST | `/v1/repair/tickets/{id}/attachments` | 上傳附件 |

### 狀態：✅ 已完成

## SRS-05-014 巡查管理

**來源**：§5-(14)

### User Story

> 身為 **GOV_STAFF**，我可建立單次或定期巡查任務；權責單位填寫巡察紀錄並派工處理。

### 主要流程

1. 建立巡查任務（任務名稱、類型：ONE_TIME/PERIODIC、區域、期程）
2. 巡查人員於現場填寫 InspectionRecord（設備、結果、備註、照片）
3. 結果為 NEED_REPAIR 時，自動觸發 E13 建立 fault_ticket
4. 異常項目可直接派工

### 資料模型

```
inspection_tasks: id, task_name, task_type(ONE_TIME/PERIODIC), 
                  area, schedule, status(ACTIVE/INACTIVE)
inspection_records: id, task_id, device_id, result(NORMAL/NEED_REPAIR/MINOR_ISSUE),
                    notes, fault_ticket_id, inspector_id
```

### 狀態：✅ 已完成
