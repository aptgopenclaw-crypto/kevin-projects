> **專案**：臺北市路燈智慧管理平台  
> **版本**：1.0  
> **日期**：2026-04-24  
> **對應**：SRS (`/03-srs/`)、WBS (`/04-wbs/`)、需求規格 (`/02-spec/`)

---

## 1. 系統範圍

臺北市路燈智慧管理平台為整合型資產管理暨維護管理系統，涵蓋路燈、分電箱、智能控制器、附掛物等照明設施之全生命週期管理，包含資產登錄、報修派工、換裝維護、材料庫存、智能監控、績效考核及行動作業。

## 3. 角色定義

| 角色代碼 | 角色名稱 | 說明 |
|---------|---------|------|
| SUPER_ADMIN | 系統管理員 | 全系統管理，帳號/角色/選單/參數設定 |
| GOV_ADMIN | 機關承辦人 | 日常業務操作、派工、審核 |
| GOV_MGR | 機關股長/分隊長 | 案件審核、進度監控 |
| GOV_CHIEF | 機關主管/首長 | 最終審核、儀表板監控 |
| OPERATOR | 專案管理/監造 | 現場監督、品質查核 |
| FIELD_USER | 現場維護人員 | 施工、回報、APP 操作 |
| CONTRACTOR | 得標廠商 | 材料查詢、派工接收、進度回報 |
| DEPT_ADMIN | 部門管理員 | 部門內帳號/權限管理 |
| PUBLIC_USER | 一般民眾 | 線上報修、進度查詢 |
| AUDITOR | 稽核人員 | 稽核日誌查詢（唯讀） |

## 6. 跨模組事件流

| 事件 | 來源模組 | 目標模組 | 觸發條件 | 動作 |
|------|---------|---------|---------|------|
| E1 | 04 障礙偵測 | 05 報修維護 | 障礙審核通過 (FAULT_REVIEW.CONFIRMED) | 自動建立 repair_ticket |
| E2 | 05 報修維護 | 03 簽核引擎 | 完工送審 | 建立 REPAIR_CLOSE 流程實例 |
| E3 | 06 換裝維護 | 03 簽核引擎 | 報竣送審 | 建立 REPLACEMENT_REVIEW 流程實例 |
| E4 | 05 報修維護 | 04 資產管理 | 派工建立 | 設備狀態 → UNDER_REPAIR |
| E5 | 08 智慧路燈 | 04 障礙偵測 | 自動故障偵測 | 建立 fault_ticket (AUTO_ALERT) |
| E6 | 06 換裝維護 | 07 材料管理 | 換裝建單 | 自動建立 issue_request |
| E7 | 07 材料管理 | 通知 | 庫存 < 安全量 | 推送低庫存預警 |
| E8 | 05 報修維護 | 07 材料管理 | 維修用料 | 扣減庫存 |
| E9 | 05 報修維護 | 04 資產管理 | 結案審核通過 | 設備 → ACTIVE + 寫 device_event |
| E10 | 06 換裝維護 | 04 資產管理 | 結案審核通過 | 舊設備 DECOMMISSIONED + 新設備 ACTIVE |
| E11 | 09 績效管理 | 05/06/08 | 計算 KPI | 讀取跨模組統計數據 |
| E12 | 07 材料管理 | 通知 | 每日排程 | 安全庫存日檢 |
| E13 | 05 巡查管理 | 04 障礙偵測 | 巡查結果=異常 | 自動建立 fault_ticket |

## 7. Function List 欄位說明

每個 SA 文件中的 Function List 採用以下統一格式：

| 欄位 | 說明 |
|------|------|
| **FN ID** | 功能編號，格式 `FN-{模組}-{序號}` |
| **功能名稱** | 功能的中文名稱 |
| **功能類型** | C=建立 / R=查詢 / U=更新 / D=刪除 / P=處理 / E=匯出 / I=匯入 / N=通知 |
| **Use Case** | 對應的使用案例編號 |
| **Actor** | 操作角色（可多個） |
| **輸入** | 功能輸入參數/資料 |
| **輸出** | 功能輸出結果/資料 |
| **商業規則** | 驗證規則、狀態限制、計算邏輯 |
| **SRS 對應** | 對應的 SRS 需求項 |
| **Spec 來源** | 對應的需求規格章節 |
| **前端頁面** | 對應的前端畫面 |
| **API 端點** | `METHOD /path` |
