# 臺北市路燈管理平台需求規格書

## 1. 基本需求

1. 使用者端操作均能透過瀏覽器執行路燈管理平台，針對分電箱設備安全檢查、燈桿設備安全檢查等，亦能以應用程式 APP 於現場執行行動化作業。
2. 除須具備機關既有路燈管理平台及應用程式 APP 之功能外，另需配合「臺北市路燈換裝暨維護案（A-B 項）」各項工作規範需求、契約期限內全程績效指標評核項目及權責異動，修正優化及新增相關所需功能，如路燈報修派工維修時效自動追蹤警示、智能路燈故障自動通報不得晚於民眾通報時間等比對功能。機關既有路燈管理平台架構、功能及使用者介面等僅供參考，廠商應以達成功能需求、提升運作效能及舒適便利性為目標。
3. 需配合資產清點及設施、管線點位 GIS 圖資之實測及回路清查，結合本府「臺北市政府財產管理系統」、工務局管線資料庫等，校修完備機關路燈資產資訊，確保資料及格式之正確性及一致性。另需建立各資產之契約資訊、保固年限等之完整維護履歷記錄及統計分析功能，藉以提升備料及預防性維護效能。
4. 歷次換裝、維護、巡（檢）查過程所要求拍攝之全部照片、影片，需儲存於平台資料庫，並能依機關指定之條件（如不同時間之同一區域或地點等），或配合平台儲存媒體重置、轉換、調整（如一定期間以前之照片備份後移出平台資料庫）等，以分類批次方式匯出至外部暫存空間，或轉存至其他媒體備份保存。另得標廠商需提供機關可合法使用之 AI 影像辨識引擎，可依據機關指定之條件（如拍照地點或相似性）將照片或含影像之檔案自動解析分類，並找出相似或重複之照片及影像。
5. 需考量未來增設各類資產設備及資料欄位之需求，可直接於介面操作新增不需額外開發，且各子系統間之數據資料需維持其完整一致性。

## 2. 系統管理

| # | 需求項目 |
|---|--------|
| (1) | **帳號管理**：包含新增、編輯、刪除帳號、修改密碼等帳號管理功能，並能依登入帳號及密碼區分各功能使用權限。帳號內容應包括使用對象區分（機關、專案管理、監造、施工及維護廠商、一般民眾等）、姓名、帳號、職務（作業角色）、在職/離職註記等相關基本資料，並須加密處理，不得以明文傳輸。如使用者帳號內容異動，不得影響既有資料內容之正確性。 |
| (2) | **人員層級及權限設定管理**：需能依機關、專案管理、監造、施工及維護廠商、一般民眾等不同使用對象，各自有至少 3 種不同層級之作業角色權限，例如機關分為承辦人、股長/分隊長、單位主管/副主管、機關首長/授權人員及系統管理員等，並可依機關需求彈性新增、修改、刪除層級或移動層級順序，以及新增或停用角色權限、設定可操作權限及權限複製功能。 |
| (3) | **登入管理**：除公開區域外，均應進行身分驗證，登入成功後始得存取。使用網頁瀏覽器時，機關、專案管理、監造、施工及維護廠商以透過實名制臺北通（掃描 QRcode）進行驗證登入為原則，且機關可使用 Taipeion 進行人員驗證及登入，如採帳號、密碼方式登入，需再輸入驗證碼或非機器人驗證。 |
| (4) | **身分驗證**：需符合「臺北市政府資通系統安全作業指引」之「身分驗證機制」規定，如密碼長度應 12 個字元以上，包含英文大小寫、數字及特殊字元，以及要求定期更換密碼等。 |
| (5) | **密碼重設機制**：忘記密碼時，可透過簡訊驗證碼或 E-mail 發送一次性及具有時效性令牌（Token），於檢查傳回令牌有效性後，始得允許使用者進行重設密碼動作。 |
| (6) | **登入逾時機制**：可設定使用者在閒置一定時間後自動登出，防止未經授權的使用者誤用或盜用。閒置時間需可調整。 |
| (7) | **登入和登出紀錄**：需可記錄使用者每次登入與登出時間，並且提供管理帳號權限者可於介面查看帳號登入登出紀錄。 |
| (8) | 其餘存取控制、識別與鑑別等相關功能應符合「資通安全責任等級分級辦法」附表十「資通系統防護基準」之「普級」規定。 |
| (9) | **系統公告發佈功能**：可供機關或其他有相關權限之人員刊登公告訊息，並可讓所有使用者於登入系統時查看公告資訊。 |
| (10) | **通知提示**：使用者登入系統後，需可即時顯示其待辦案件、列管案件之提示畫面，並可直接點選查閱負責案件之新增、審核、結案等進度相關資訊，並直接可執行簽核或抽回。 |

| # | 需求項目 |
|---|--------|

## 4. 資產管理

基本上需具備地理資訊、資產資訊、管理資訊、附掛物資訊，符合路燈維護工作及績效指標評核需求，並配合 GIS 圖資應用以視覺化（Visualization）呈現資產資料。

### 4.1 地理資訊

#### A. 路燈地圖

以圖形化（Graphical）介面呈現路燈、分電箱點位及管線位置。精確圖資採用 ArcGIS 圖資平台建置及更新，但行動運用瀏覽則轉置於開放式圖台（Google Map、Bing Map 或 Open Layer）。電腦螢幕單一畫面需能顯示至少任 1 張千分之一地形圖上（或 Google Map 縮放等級 15）全數之路燈、分電箱點位及管線位置。

#### B. 坐標資料整合

需包含但不限於：
- 1997 臺灣大地基準（TWD97）二度分帶坐標
- 經緯度（GPS）坐標
- 台電公司「電力坐標」（TWD67 TM2 投影坐標編碼）

且具備坐標轉換功能。

#### C. 街景及圖資介接

需可連結到實際街景（如 Google Map 街景）查閱設施周邊狀況，並可依機關需求介接相關地理圖資資訊（包含但不限於內政部國土署、Google Map 街道圖、GIS 管線圖、道路及人行道視覺化平台）。若有圖資介接申請需求，由機關協助得標廠商申請；若因地理資訊連接街景及圖資衍生費用，由得標廠商負擔。

#### D. 基本圖資

- **(A)** 圖例、符號說明及管理。圖例、符號需產製輸出為電腦繪圖軟體可使用之向量圖元。
- **(B)** 地形圖底圖：TWD97 臺北市一仟分之一數值地形圖。
- **(C)** 分區範圍管理：行政區里、機關分隊分區圖、台電區處及包燈電號分區圖、換裝維護廠商分區圖等。
- **(D)** 機關路燈標準圖：年度更新修訂，可用連結處理。

#### E. 坐標資訊

- **(A)** 坐標系統平面基準採用 TWD97 二度分帶，高程基準採用 TWVD2001，資料規範及屬性依據內政部「國土資訊系統－公共設施管線資料庫標準制度」、「公共設施管線資料標準」及本府工務局規定之格式辦理。
- **(B)** 需包含路燈及分電箱點位、管線三維坐標位置（含架空管線及回路綁定）。
- **(C)** 需可差異化呈現平面/高架道路、隧道/車行地下道等不同道路型態。

#### F. 圖資匯入、匯出

- **(A)** 匯出、匯入時需比對確認內容之正確性。匯入內容如與既有資料有所差異，應經檢視確認後才可進行更動。
- **(B)** 需可匯出本府工務局規定之地理標誌語言（Geography Markup Language，GML）格式圖資。
- **(C)** 需可匯出「臺北市資料大平臺」規定格式之「臺北市路燈位置分布圖」及「臺北市路燈迴路開關箱資料」圖資資料。
- **(D)** 自工務局（道管中心）管線資料庫下載之圖資需可匯入機關路燈管理平台。
- **(E)** 需可匯出指定區域之圖資資料（含坐標向量）至電腦繪圖軟體使用，除匯出路燈及分電箱點位、管線向量及基本資料外（含圖例、符號之向量圖元），需可設定輸出圖資比例尺及圖面尺寸（如 1/1000，A3 紙張尺寸）。
- **(F)** 需可整合各單位及列管建案基地施工範圍、應辦事項資訊，如機關預定/施工中新設、整建路燈點位、建案周邊路燈臨拆復舊、遷移或新設、臺北市政府工務局道路挖掘管理中心、台電施工資訊等。

### 4.2 管理資訊

| # | 需求項目 |
|---|--------|
| A | 需包含預算年度、採購案號、契約名稱、契約廠商、對應資產類別、數量、契約期限、驗收日期與保固年限等相關資訊。需可記錄歷史資料，依需求新增、編輯、刪除、停用契約資訊，並可彈性增加資料欄位及內容。 |
| B | 資產明細需與所屬之契約名稱、財物分類及明細等連結，以供後續維修時查詢使用、保固年限等相關資訊。 |
| C | 需有預防性維護、定期檢查（汰換）之分區、分期、周期或日期等編輯管理及時間提醒功能。 |
| D | 需可依契約名稱、採購案號等篩選出對應之資產清單，以批次方式轉移該資產至所屬契約，以適用契約保固維護到期後之維護權責轉移至他案功能。 |
| E | 得標廠商需定義契約管理所需資料欄位格式，以便機關彙整相關清冊以供匯入系統。若其中有屬資料庫主鍵（Primary Key）或非空欄位（Not Null Columns）之資料，而機關未有該項資料、資料不全或格式不符時，得標廠商需協助機關給定預設值（Default Values）並提供補正方案。 |
| F | 需能提供契約資料匯出功能，匯出檔案可為開放文件試算表格式（.ods）、Microsoft Excel 試算表格式（.xls）或 CSV 格式。 |

### 4.3 資產資訊

#### A. 燈桿設備

包含路燈號碼牌編號、系統編號（平台建置流水號）、坐標位置（TWD97 二度分帶坐標及經緯度坐標、台電坐標）、設置（更新）年月、財產歸屬、財物分類及明細（含「臺北市政府財產管理系統」要求項目）、設置位置（AC 路面、人行道/標線人行道，有無符合最小淨寬規定、張貼反光紙等）、基礎型式（基座式、螺栓式、溝上式等）、歷來整建及維護契約名稱、權責維護期限、廠商資訊、換裝歷程（含當時換裝照片）、報修歷程（含當時維修照片）等資產維護及認養、認捐歷程紀錄等。

#### B. 燈桿屬性

包含：
- 材質（如不鏽鋼、鍍鋅鋼板、鋁桿、木桿、水泥桿）
- 型態（自備桿、附掛台電桿、附掛外牆等）
- 樣式（單臂、雙臂、雙臂-高低燈、多臂、造型、吸頂、壁掛等）
- 高度
- 照明分類（道路、人行道、隧道、公園等）

#### C. 燈具資料

包含編號、規格（含廠牌、型號、額定/實際光通量、額定/實際功率、色溫、標稱壽命、出廠年月、燈具 ID、安裝日期、保固期限）、光源類別（LED、高壓鈉燈、複金屬燈等）、控制型態（智能、非智能、定時、光控感應、其他控制方式等）、引接之電源分電箱、回路編號、台電電號與用電類別（包燈、表燈）、智能控制器規格（含廠牌、型號、出廠年月、控制器 ID、電信商與 IP、安裝日期、保固期限等）。

#### D. 分電箱設備

包含分電箱編號、坐標位置（TWD97 二度分帶坐標及經緯度坐標）、設置（更新）年月、類型、裝置高度（有無達 1.5 公尺）、設置位置（AC 路面、人行道/標線人行道，有無符合最小淨寬規定等）、開關箱安全檢查紀錄、財產歸屬、台電電號、廠商資訊、歷來整建及維護契約名稱、權責維護期限等。

#### E. 電力設備

如高低壓、UPS、發電機：參考分電箱設備內容辦理，並含個別設施內容等資訊。

#### F. 其他智能相關設備、附掛物或附掛管線

如風光互補、智慧園燈、車流偵測系統等：參考燈桿設備、燈桿屬性、燈具資料及分電箱設備辦理，並含個別設施內容等資訊。如屬他單位附掛物，需包含申設單位、核准設置日期、文號、設置（租賃）期限等資料。

#### G. 圖形化搜尋

需提供於圖形化介面或清冊搜尋時，能以辨識圖示或篩選相關欄位資料方式快速找出所需資產資訊。

### 4.4 資產異動

需有資產異動編輯功能，包含但不限於單筆或批次加帳、除帳及變更，需有機關審核流程、應備文件等。

### 4.5 統計報表

資產管理需具備統計報表功能，所有資產清冊均可提供全欄位資料匯出功能，匯出檔案可為開放文件試算表格式（.ods）、Microsoft Excel 試算表格式（.xls）或 CSV 格式。

### 4.6 工程驗收接管

需有工程驗收接管資料檔案管理功能，包含但不限於竣工圖、設備規格、操作維護手冊等，可以連結方式節省主機空間。

## 5. 報修維護管理

需配合績效指標評核需要，如故障報修案件自立案到維護完工回報，能自動計算維護時數做為評分依據，若案件被退回補件，亦能自動延長及追蹤；若係智能路燈自動通報，需有比對不得晚於民眾通報等功能。

| # | 需求項目 |
|---|---------|
| (1) | 需提供公開 Web 網頁及掃描現場路燈號碼牌 QR Code 連結 2 種方式，比照「臺北市陳情系統」實名制模式，供民眾線上報修及查詢報修案件處理進度等。報修網頁須有個資使用聲明，並可上傳限定件數、容量之報修照片、錄影、錄音附件。 |
| (2) | 需可介接外部單位通報系統整合立案，包含臺北市陳情系統 1999 等機關指定介接之平台。 |
| (3) | 收受報修案件或回報維修處理情形時，須自動檢查上傳之附件有無夾帶病毒或惡意程式碼。 |
| (4) | 需具備各案得標廠商派工及維護人員線上收件、人工立案、回報維修處理情形（維修前、中、後照片需記錄日期、時間、GPS 位置）、匯出案件清冊及報表分析等功能。 |
| (5) | 需提供派工等相關人員查詢案件狀態、審核維護案件、匯出案件清冊及報表分析等功能。 |
| (6) | 維護案件狀態包含但不限於：未收案、已派工、待處理、完工審核中、退回補件、改分（轉送）、追蹤中等，需能區分派工及維護人員權責方，且每次狀態轉換均需有歷程紀錄，可查看各狀態更新之執行帳號、執行時間及修改前後異動紀錄。 |
| (7) | **維護案件記錄內容**需包含但不限於下列項目： |
|   | A. 報修資訊：包含報修時間、報修人姓名、聯絡及回復方式、報修地點、報修項目、報修附件等。 |
|   | B. 維護資訊：包含故障原因（類別）、換裝之資產資訊（如燈具）、維護完成時間、維護前中後照片等。 |
| (8) | 檢視報修案件（瞭解概況）時，平台需提供報修路燈之基本資料、歷來維護履歷、位置地圖檢視功能，並可依維護權責派工給維護單位或契約廠商。 |
| (9) | 查詢路燈資訊時，需可查看歷史維護案件紀錄，並可檢視每次維護內容詳情、設備更換內容及設備剩餘可用、保固或維護年限。 |

| (10) | 換裝案件須有相關審核流程，包含完工後自主檢核、報竣審核、結案審核等機制。若經審退，需有更正補件重送機制；已結案案件被查核發現有錯誤時，須具備補救機制，以保持資產資料之正確性。 |

| (12) | 可依篩選條件匯出派工案件清冊及異動之路燈地圖，並可產出竣工清單或用電申請地點表供竣工查核（含環境清潔、圖資修正校核等）或受理用電變更。 |

## (六) 更新建置效能需求

得標廠商更新建置之路燈管理平台，除涉及大量彙算或印表作業外：

- 在 **100 個**使用者同時作業尖峰時段，測試其回應時間 **95% 須以不超過 5 秒**為原則
- 應用程式回應時間以**不超過 1 秒**為原則
- 程式內處理繁雜，經機關同意者不在此限
