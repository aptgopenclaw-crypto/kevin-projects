## 3. 簽核管理

| # | 需求項目 |
|---|--------|
| (1) | 需陳核之案件或表單，需以**表列與圖示**二種方式呈現工作流程及狀態，讓承辦人及相關人員可以隨時檢視案件之處理狀態，以及經過的每一步驟與時間點。 |
| (2) | **代理人設定**：提供案件處理角色代理分派指定作業，代理人於代理期間於系統主畫面即可辦理被代理人案件。 |

## 5. 報修維護管理

需配合績效指標評核需要，如故障報修案件自立案到維護完工回報，能自動計算維護時數做為評分依據，若案件被退回補件，亦能自動延長及追蹤；若係智能路燈自動通報，需有比對不得晚於民眾通報等功能。

| # | 需求項目 |
|---|---------|
| (1) | 需提供公開 Web 網頁及掃描現場路燈號碼牌 QR Code 連結 2 種方式，比照「臺北市陳情系統」實名制模式，供民眾線上報修及查詢報修案件處理進度等。報修網頁須有個資使用聲明，並可上傳限定件數、容量之報修照片、錄影、錄音附件。 |
| (2) | 需可介接外部單位通報系統整合立案，包含臺北市陳情系統 1999 等機關指定介接之平台。 |
| (3) | 收受報修案件或回報維修處理情形時，須自動檢查上傳之附件有無夾帶病毒或惡意程式碼。 |
| (4) | 需具備各案得標廠商派工及維護人員線上收件、人工立案、回報維修處理情形（維修前、中、後照片需記錄日期、時間、GPS 位置）、匯出案件清冊及報表分析等功能。 |
| (5) | 需提供派工等相關人員查詢案件狀態、審核維護案件、匯出案件清冊及報表分析等功能。 |
| (6) | 維護案件狀態包含但不限於：未收案、已派工、待處理、完工審核中、退回補件、改分（轉送）、追蹤中等，需能區分派工及維護人員權責方，且每次狀態轉換均需有歷程紀錄，可查看各狀態更新之執行帳號、執行時間及修改前後異動紀錄。 |
| (7) | **維護案件記錄內容**需包含但不限於下列項目： |
|   | A. 報修資訊：包含報修時間、報修人姓名、聯絡及回復方式、報修地點、報修項目、報修附件等。 |
|   | B. 維護資訊：包含故障原因（類別）、換裝之資產資訊（如燈具）、維護完成時間、維護前中後照片等。 |
| (8) | 檢視報修案件（瞭解概況）時，平台需提供報修路燈之基本資料、歷來維護履歷、位置地圖檢視功能，並可依維護權責派工給維護單位或契約廠商。 |
| (9) | 查詢路燈資訊時，需可查看歷史維護案件紀錄，並可檢視每次維護內容詳情、設備更換內容及設備剩餘可用、保固或維護年限。 |
| (10) | 現場維護處理結果，需通報機關或機關必須知悉者，或應由機關納入附約執行或另行招商辦理事項之案件，需可自動產生通知機關報告單、查報單及後續之設計單。報告單、查報單及設計單需具備繪圖編輯功能（如呼叫 ArcGIS 或 AutoCAD 物件），可帶入路燈、分電箱及管線之點位坐標、套疊地形圖、路燈圖資及淡化或去除地形圖不需要之標記、圖元，並可於結案審核完成後匯入平台更新圖資。 |
| (11) | 維護案件處理情形須有相關審核流程，包含維護完成送審、結案審核、退回更正、補件重送等機制，已結案之案件亦須可經過結案審核退回再次檢修或重新更正處理情形。完整之審核機制與流程，以機關規定為準。 |
| (12) | 維修案件完工送審後，平台需能自動同步更新資產資訊，若遇退回補件，亦可由送審人員選擇是否再次同步更新資產，以確保期間無其他案件更新資產，導致新資料被舊資料覆蓋等問題。 |
| (13) | 需具備統計報表功能，可依需求篩選條件匯出報表，其資訊包含但不限於維護案件統計、維護路燈時間統計、通報來源統計、故障類別統計、故障原因、故障熱區、材料換修數量統計等。報表需提供全欄位資料匯出功能，匯出檔案可為 .ods、.xls 或 CSV 格式。 |
| (14) | 需符合機關巡查及相關機關督導、稽核需求，可新增/編輯單次或定期巡查項目、期程、區域等工作，並可由相關權責單位填寫巡察紀錄、上傳照片及相關附件後，派工交付廠商處理。 |
| (15) | 需提供篩選條件或流程，以迅速查看非屬契約維護案件、附約維護案件等資料，作為資料分析、快速匯出及相關派工作業。 |
| (16) | 需可匯出「臺北市資料大平臺」規定格式之「臺北市路燈維修資料」。 |
| (17) | 需有適當方式（如電子郵件或平台帳號）可提供里辦公處（里長）收取或查詢其里內路燈故障處理狀況。 |

## 6. 換裝維護管理

| # | 需求項目 |
|---|---------|
| (1) | **路燈號碼牌編號管理**：需依機關訂定之編號規則產出不重複之編號，並可依需求產出（重製）對應之路燈號碼牌編號及連結報修網頁之 QRCode，供製作張貼（補貼）至燈桿上。 |
| (2) | **路燈資產清冊匯入功能**：需可配合各案得標廠商之路燈資產普查工作，以批次匯入方式一次更新普查資訊。普查資訊內容需符合資產管理需求，如分電箱編號（本案新增編號）、詳細地址、地點描述、台電桿號（附掛台電桿）、換裝前後燈種及瓦數等。 |
| (3) | **資產異動管理功能**：需能以案件管理方式，清楚掌握各路燈、分電箱等各項資產資料異動紀錄。 |
| (4) | **材料清單匯入功能**：需可使用批次上傳方式匯入經審驗合格之燈具、智能路燈控制器等材料清單。清單欄位須符合材料管理需求，包含契約名稱、燈具/控制器編號、審驗合格日期、生產批號、廠牌、型號等資訊，燈具則尚需包含型號、色溫、額定光通量、額定電壓、額定電流、額定功率等。 |
| (5) | 需能管控維護或換裝時，必須使用經審驗合格且平台已建置材料編號之設備（如燈具、智能路燈控制器、燈桿號碼牌等）始得更新資產資料。 |
| (6) | **派工管理功能**：可依契約要求開立派工單，填寫派工事由、案件類別、地點、數量、工期等所需內容，並可指派給對應之維護廠商。維護廠商亦需以派工單或其他方式管控換裝維護進度。 |
| (7) | **派工案件類別管理**：可依機關需求彈性調整，包含但不限於新設（補裝）、換裝（更換規格）、遷移、停用（廢止）、調整方向或角度、加裝遮光罩等。 |
| (8) | 新設（補裝）、遷移路燈案件需可預先匯入普查完成之路燈編號與地理位置資訊，並提供地圖查詢功能便於施工作業時辨別應裝設、遷移路燈之位置。 |
| (9) | **廠商自主檢核機制**：換裝案件經廠商自主審核完成即可預先更新系統相關資產資訊（亦即自主檢核完成後便可於路燈地圖或清冊查看最新資產內容）。 |
| (10) | 換裝案件須有相關審核流程，包含完工後自主檢核、報竣審核、結案審核等機制。若經審退，需有更正補件重送機制；已結案案件被查核發現有錯誤時，須具備補救機制，以保持資產資料之正確性。 |
| (11) | **統計報表功能**：需可查看各契約之派工案件類型、已交付件數與實際完成件數、分類累積數量、執行進度百分比等。 |
| (12) | 可依篩選條件匯出派工案件清冊及異動之路燈地圖，並可產出竣工清單或用電申請地點表供竣工查核（含環境清潔、圖資修正校核等）或受理用電變更。 |

## 9. 績效管理

### (1) 績效指標自動統計

需能依績效指標評核項目自動記錄、統計及結算各項成果績效分數，包含但不限於下列項目：

| 項目 | 說明 |
|------|------|
| A | 績效指標評核項目、計算公式及條件設定之新增、修改、刪除等管理功能。 |
| B | 可依績效指標計算需求，單筆或整批匯入計分所需數據資料。 |
| C | 依平台紀錄或輸入資料，計算各期本案各項績效指標服務水準及績效，並於介面呈現績效統計分數及提供匯出應檢附文件之報表功能（如加、扣點資料、估驗計價等）。 |
| D | 需提供鎖定功能，一經鎖定則固定該期績效計分內容，以避免因資料更新而造成計分變動情形。 |
