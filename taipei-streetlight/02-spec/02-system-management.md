# 02 系統管理

> 來源：需求 2-(1) ~ 2-(10) ｜預估工時：26

## 功能清單

### 2-1 帳號管理 `工時 5`
- 新增、編輯、刪除帳號、修改密碼
- 依登入帳號區分功能使用權限
- 帳號內容：
  - 使用對象區分（機關、專案管理、監造、施工及維護廠商、一般民眾）
  - 姓名、帳號、職務（作業角色）、在職/離職註記
- **安全要求**：加密處理，不得明文傳輸
- 帳號內容異動不得影響既有資料正確性
- **現狀**：✅ 已完成（UserAdmin CRUD、角色指派、停用/軟刪除）

### 2-2 人員層級及權限設定管理 `工時 5`
- 不同使用對象各自至少 3 種不同層級之作業角色權限
- 機關層級範例：承辦人、股長/分隊長、單位主管/副主管、機關首長/授權人員、系統管理員
- 功能需求：
  - 彈性新增、修改、刪除層級
  - 移動層級順序
  - 新增或停用角色權限
  - 設定可操作權限
  - **權限複製**功能
- **現狀**：✅ 已完成（RBAC + DataScope + 角色管理 + 選單管理）
- **待補**：權限複製功能

### 2-3 登入管理 `工時 5`
- 公開區域外均需身分驗證
- 網頁端：
  - 機關、專案管理、監造、廠商 → 透過**臺北通**（掃描 QRCode）驗證登入
  - 機關可使用 **Taipeion** 人員驗證
  - 帳號密碼方式 → 需再輸入**驗證碼**或非機器人驗證
- **現狀**：✅ 已完成（JWT + 驗證碼登入）
- **待補**：臺北通 QRCode 整合、Taipeion 整合

### 2-4 身分驗證 `工時 2`
- 符合「臺北市政府資通系統安全作業指引」之「身分驗證機制」
- 密碼規則：
  - 長度 ≥ 12 字元
  - 包含英文大小寫、數字及特殊字元
  - 要求定期更換密碼
- **現狀**：✅ 已完成（密碼規則 ≥ 8 字元）
- **待補**：密碼長度調整為 12 字元、定期更換密碼提醒

### 2-5 密碼重設機制 `工時 2`
- 忘記密碼時：
  - 透過**簡訊驗證碼**或 **E-mail** 發送一次性且具時效性令牌（Token）
  - 檢查傳回令牌有效性後始得重設密碼
- **現狀**：❌ 未實作

### 2-6 登入逾時機制 `工時 1`
- 使用者閒置一定時間後**自動登出**
- 閒置時間需可調整
- **現狀**：✅ 已完成（前端閒置偵測 + 倒數警告 Dialog + 自動登出 + 後台設定頁可調整閒置時間 + 審計事件 `IDLE_TIMEOUT_LOGOUT`）
- **設計文件**：`x-plan/2-6-登入預期機制.md`

### 2-7 登入和登出紀錄 `工時 1`
- 記錄使用者每次登入與登出時間
- 管理帳號權限者可於介面查看帳號登入登出紀錄
- **現狀**：✅ 已完成（Audit History，含 USER_AUTH 事件）

### 2-8 資安合規 `工時 0`
- 存取控制、識別與鑑別等功能應符合：
  - 「資通安全責任等級分級辦法」附表十「資通系統防護基準」之「普級」規定
- **現狀**：⚠️ 需逐項檢核

### 2-9 系統公告發佈 `工時 2`
- 機關或有權限人員可刊登公告訊息
- 所有使用者登入時可查看公告資訊
- **現狀**：✅ 已完成（公告 CRUD、草稿/發佈/過期狀態、全體/指定部門受眾、置頂、排程發佈、鈴鐺通知、公告欄、已讀追蹤）
- **設計文件**：`x-plan/2-9-系統公告發佈.md`


### 2-10 通知提示 `工時 3`
- 登入後即時顯示：待辦案件、列管案件提示畫面
- 可直接點選查閱案件進度（新增、審核、結案等）
- 可直接執行簽核或抽回
- **現狀**：❌ 未實作（需搭配簽核管理模組）
