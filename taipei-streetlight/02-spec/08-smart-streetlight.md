# 08 智能路燈管理

> 來源：需求 8-(1) ~ 8-(10) ｜預估工時：30
> 涵蓋：智能路燈、智慧電表、智慧園燈

## 功能清單

### 8-1 IoT 數據串接平台
- 依經濟部「**5G 智慧杆系統技術規範**」制訂串接方案
- 整合本市既有及未來新增之智能路燈數據
- 定義統一之 **API**，讓全市智能路燈及相關 IoT 裝置上下行溝通
- 需提出方案：
  - 降低串接時數據資料錯漏
  - 發生時快速辨析釐清權責方
  - 快速補救方案

### 8-2 即時設備狀態地圖
- 於路燈地圖中查看即時設備狀態
- 以不同圖示顏色區分：
  - 離線、熄燈、亮燈（全亮）、調光、故障、維修中

### 8-3 自動故障偵測與告警
- 依設備回傳訊號自動判斷故障類別及告警
- 報表或圖表形式顯示與匯出
- 可在平台設定相關條件
- 用於自動報修或人工報修

### 8-4 智慧電表整合
- 整合智慧電表與智能路燈回傳訊號
- 針對**區域性停電**或**電源回路跳脫**等故障原因
- 自動判斷故障類別及告警

### 8-5 調光控制
- 透過平台介面整合控制所有智能路燈
- 範圍：本案及現有他案維護保固中之所有智能路燈
- 功能：
  - **單燈**或**群組**調光（點滅）
  - 即時調光
  - 排程調光（單次或重複週期）
- 安全設計：
  - **失效安全（Fail-Safe）**
  - 考量包燈用電非 24 小時連續供電
  - 考量網路傳輸延遲/中斷
  - 考量智能路燈控制器記憶功能

### 8-6 連線紀錄
- 查看設備最後一次連線時間
- 即時設備狀態
- 最近 7 天連線 Log 紀錄
- 報表或圖表形式顯示與匯出

### 8-7 示警設定
- 提供操作介面設定調整：
  - 偵測日間亮燈時段
  - 夜間異常時段
  - 回傳訊號中斷時間
  - 設備電力變化
- 可依不同**區域、路段**進行個別設定

### 8-8 績效自動計算
- 配合績效指標定義及計算公式
- 自動計算運作維護之績效與得分
- 依賴：績效管理模組（09）

### 8-9 統計分析
- 智能路燈用電統計
- 自動告警統計
- 故障統計

### 8-10 隧道/地下道監控
- 隧道、地下道照明系統納入監視及異常告警
- 遠端控制需有資安防護機制
- **失效安全（Fail-Safe）** 設計

## 核心實體

| 實體 | 說明 |
|------|------|
| `iot_devices` | IoT 設備註冊（控制器、電表等） |
| `device_status` | 設備即時狀態 |
| `device_telemetry` | 設備回傳訊號數據 |
| `device_alerts` | 告警紀錄 |
| `alert_rules` | 告警規則設定 |
| `dimming_schedules` | 調光排程 |
| `dimming_commands` | 調光指令紀錄 |
| `connection_logs` | 設備連線紀錄 |

## 技術考量

| 項目 | 方案 |
|------|------|
| IoT 協定 | MQTT / CoAP / HTTP REST |
| 即時狀態 | WebSocket 推送至前端 |
| 時序數據 | TimescaleDB 或 InfluxDB |
| 調光指令 | 非同步佇列 + 重試機制 |
| 失效安全 | 控制器本地排程 + 平台排程雙重保障 |
