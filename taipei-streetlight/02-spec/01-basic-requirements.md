# 01 基本需求與架構原則

> 來源：需求 1-(1) ~ 1-(5)

## 核心約束

### 1-1 瀏覽器 + APP 雙端
- 路燈管理平台所有使用者端操作均透過瀏覽器執行
- 分電箱設備安全檢查、燈桿設備安全檢查等，需以 APP 於現場執行行動化作業

### 1-2 既有功能延伸 + 新增需求
- 須具備機關既有路燈管理平台及 APP 功能
- 需配合「臺北市路燈換裝暨維護案（A-B 項）」各項工作規範
- 新增功能範例：
  - 報修派工維修時效**自動追蹤警示**
  - 智能路燈故障自動通報**不得晚於民眾通報**時間比對功能

### 1-3 資產清點與 GIS 圖資整合
- 結合「臺北市政府財產管理系統」、工務局管線資料庫
- 校修完備路燈資產資訊（資料及格式正確性、一致性）
- 建立契約資訊、保固年限之完整維護履歷記錄及統計分析

### 1-4 照片影片管理 + AI 影像辨識
- 歷次拍攝之所有照片、影片需儲存於平台資料庫
- 支援依指定條件分類批次匯出至外部暫存空間或轉存備份
- **AI 影像辨識引擎**：
  - 依拍照地點或相似性自動解析分類
  - 找出相似或重複之照片及影像

### 1-5 擴充性設計
- 未來增設各類資產設備及資料欄位時，可直接於介面操作新增
- 不需額外程式開發
- 各子系統間之數據資料需維持完整一致性

## 技術影響

| 約束 | 技術決策 |
|------|---------|
| 雙端（Web + APP） | 後端 RESTful API 設計、前端 Vue SPA + 原生/跨平台 APP |
| GIS 圖資 | ArcGIS 精確圖資 + 開放式圖台（Google Map / OpenLayers）瀏覽 |
| AI 影像辨識 | 需整合合法授權之 AI 引擎（獨立服務或雲端 API） |
| 動態欄位 | 資產設備需支援自定義欄位（EAV 或 JSON Schema 設計） |
| 資料一致性 | 跨子系統事務一致性、事件驅動同步 |
