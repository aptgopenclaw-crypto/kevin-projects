# 05 報修維護管理

> 來源：需求 5-(1) ~ 5-(17) ｜預估工時：30

## 功能清單

### 5-1 民眾線上報修
- 公開 Web 網頁報修
- 掃描現場路燈號碼牌 **QR Code** 連結報修
- 比照「臺北市陳情系統」**實名制**模式
- 報修網頁須有個資使用聲明
- 可上傳限定件數、容量之報修照片、錄影、錄音附件

### 5-2 外部系統通報整合
- 介接外部通報系統整合立案
- 包含：臺北市陳情系統 **1999** 等機關指定介接之平台

### 5-3 上傳附件安全掃描
- 收受報修案件或回報時，自動檢查上傳附件有無夾帶**病毒或惡意程式碼**

### 5-4 派工及回報
- 各案得標廠商派工
- 維護人員線上收件、人工立案
- 回報維修處理情形
  - 維修前、中、後照片需記錄**日期、時間、GPS 位置**
- 匯出案件清冊及報表分析

### 5-5 案件查詢與審核
- 相關人員查詢案件狀態
- 審核維護案件
- 匯出案件清冊及報表分析

### 5-6 案件狀態管理
- 狀態包含但不限於：
  - 未收案、已派工、待處理、完工審核中、退回補件、改分（轉送）、追蹤中
- 區分派工及維護人員權責方
- 每次狀態轉換需有歷程紀錄：
  - 執行帳號、執行時間、修改前後異動紀錄

### 5-7 維護案件記錄
- **報修資訊**：報修時間、報修人姓名、聯絡方式、報修地點、報修項目、附件
- **維護資訊**：故障原因（類別）、換裝資產資訊、維護完成時間、維護前中後照片
- 詳細欄位依需求訪談後之紀錄及需求清單為準

### 5-8 報修案件概覽
- 檢視報修案件時提供：
  - 報修路燈基本資料
  - 歷來維護履歷
  - 位置地圖檢視
- 可依維護權責派工給維護單位或契約廠商

### 5-9 歷史維護紀錄
- 查詢路燈資訊時可查看歷史維護案件
- 可檢視每次維護內容詳情
- 設備更換內容及剩餘可用/保固/維護年限

### 5-10 自動產生通報單
- 通報機關之案件自動產生：
  - 通知機關報告單
  - 查報單
  - 設計單
- 具備**繪圖編輯功能**（呼叫 ArcGIS 或 AutoCAD 物件）
- 可帶入路燈、分電箱及管線之點位坐標
- 結案審核完成後匯入平台更新圖資

### 5-11 審核流程
- 維護完成送審 → 結案審核 → 退回更正 → 補件重送
- 已結案案件可經退回再次檢修或重新更正
- 完整審核機制依機關規定
- 依賴：簽核管理模組（03）

### 5-12 完工自動同步資產
- 維修案件完工送審後，自動同步更新資產資訊
- 退回補件時，送審人員可選擇是否再次同步
- 避免新資料被舊資料覆蓋

### 5-13 統計報表
- 可依需求篩選條件匯出報表
- 統計項目：
  - 維護案件統計、維護時間統計
  - 通報來源統計、故障類別統計
  - 故障原因、故障熱區
  - 材料換修數量統計
- 匯出格式：ODS / XLS / CSV

### 5-14 巡查管理
- 新增/編輯單次或定期巡查項目、期程、區域
- 權責單位填寫巡察紀錄、上傳照片及附件
- 派工交付廠商處理

### 5-15 非契約/附約案件篩選
- 提供篩選條件或流程，迅速查看非契約/附約案件
- 快速匯出及派工作業

### 5-16 開放資料匯出
- 匯出「臺北市資料大平臺」規定格式之「臺北市路燈維修資料」

### 5-17 里辦公處通知
- 提供方式（電子郵件或平台帳號）
- 里長可收取或查詢里內路燈故障處理狀況

## 核心實體

| 實體 | 說明 |
|------|------|
| `repair_tickets` | 報修案件主表 |
| `repair_status_log` | 案件狀態變更歷程 |
| `repair_attachments` | 案件附件（照片/影片/錄音） |
| `repair_dispatches` | 派工紀錄 |
| `inspection_tasks` | 巡查任務 |
| `inspection_records` | 巡查紀錄 |
