# 03 簽核管理

> 來源：需求 3-(1) ~ 3-(2) ｜預估工時：15

## 功能清單

### 3-1 案件工作流程及狀態視覺化 `工時 10`
- 需陳核之案件或表單，以**表列**與**圖示**二種方式呈現工作流程及狀態
- 承辦人及相關人員可隨時檢視：
  - 案件處理狀態
  - 經過的每一步驟與時間點
- **適用範圍**：
  - 報修維護案件審核
  - 換裝維護案件審核
  - 資產異動審核
  - 其他需陳核之表單

### 3-2 代理人設定 `工時 5`
- 提供案件處理角色之**代理分派指定**作業
- 代理人於代理期間，可在系統主畫面辦理被代理人案件
- 需求：
  - 指定代理人與代理期間
  - 代理期間待辦自動轉派
  - 代理期間結束自動回復

## 核心設計

### 工作流程引擎
- 需支援多種案件類型，每種有不同的簽核流程
- 流程節點：送審 → 審核 → 退回/通過 → 結案
- 每個節點需記錄：執行帳號、執行時間、修改前後異動

### 資料模型

| 實體 | 說明 |
|------|------|
| `workflow_definitions` | 流程定義（案件類型 → 步驟序列） |
| `workflow_instances` | 流程實例（與具體案件綁定） |
| `workflow_steps` | 流程步驟紀錄（每一步驟操作歷程） |
| `delegate_settings` | 代理人設定（代理人、被代理人、期間） |

### 跨模組依賴
- 報修維護管理（模組 5）：維護案件審核流程
- 換裝維護管理（模組 6）：換裝案件審核流程
- 資產管理（模組 4）：資產異動審核流程
- 系統管理 2-10：通知提示、待辦案件
