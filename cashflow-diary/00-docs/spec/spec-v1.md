# 📱 CashFlow Diary (現金流日記) — v1 規格書

## 一、 產品定位與核心原則
*   **產品定位**：一款專注於「日常小額支出」的純本地端記帳工具，透過高頻率的記錄與週/月維度的統計，幫助使用者掌握現金流去向。
*   **目標平台**：Android 專用（最低 SDK 版本待定，建議 API 26+）
*   **核心原則**：
    1.  **Zero Friction (零摩擦)**：無須登入、無須等待網路載入，打開 APP 3 秒內完成記帳。
    2.  **Local First (本地優先)**：所有數據 100% 儲存於手機本地，保障隱私。
    3.  **Data Driven (數據驅動)**：強調週/月統計的直觀性，讓數字說話。

---

## 二、 核心功能模組規劃

### 1. 快速記帳模組 (Input)
*   **計算機鍵盤**：進入記帳頁直接彈出數字鍵盤，支援小數點與退格，符合直覺。
*   **智能預設**：
    *   日期預設為「今天」，時間預設為「當前時間」（可點擊修改）。
    *   分類預設為「上次使用的分類」或「預設分類」。
    *   幣別預設為本位幣（預設 TWD）。
*   **欄位設計**：
    *   **金額**（必填）
    *   **分類**（必填，從預設清單選擇）
    *   **幣別**（選填，點擊可切換常用幣別：TWD、RMB、USD、JPY 等）
    *   **備註**（選填，如：與同事吃午餐）
    *   **標籤**（選填，如：#可抵稅、#公司報帳，方便後續篩選）
*   **連續記帳模式**：儲存後不跳轉頁面，僅清空金額，方便一次結算當天多筆小額消費。

### 2. 明細與管理模組 (Manage)
*   **時間軸列表**：首頁以「日期」為單位折疊展示（如：今天、昨天、本週三）。
*   **快捷操作**：
    *   左滑單筆紀錄：顯示「刪除」與「修改」按鈕。
    *   點擊單筆紀錄：進入編輯頁。
*   **搜尋與篩選**：支援依「關鍵字（備註）」、「分類」、「特定日期區間」進行篩選。
*   **今日/本月小計**：列表頂部懸浮顯示「今日總支出」與「本月總支出」。

### 3. 統計與分析模組 (Analyze) —— 🌟 核心需求
此模組分為「週統計」與「月統計」兩個主要 Tab，並提供視覺化圖表。

#### 📊 A. 週統計 (Weekly Stats)
*   **時間定義**：支援切換「本週 (週一至週日)」、「上週」、或「過去 7 天」。
*   **數據看板**：
    *   **本週總支出**（大字體顯示，並附帶與上週同期的增減百分比，如：🔺 15%）。
    *   **每日花費長條圖**：X 軸為週一至週日，Y 軸為金額，一眼看出哪天花最多。
*   **分類佔比**：圓餅圖 (Pie Chart) 顯示本週各分類佔比，點擊圖例可過濾明細。

#### 📈 B. 月統計 (Monthly Stats)
*   **時間定義**：支援切換「本月」、「上月」，或自訂月份。
*   **數據看板**：
    *   **本月總支出**（附帶與上月同期的增減百分比）。
    *   **預算進度條**：顯示本月預算（預設 NT$15,000）的消耗進度，超支時顯示紅色警告。
    *   **花費趨勢折線圖**：X 軸為 1 號至 30/31 號，展示每日花費的波動；或改為「每週花費長條圖」展示月度節奏。
*   **分類排行榜**：條形圖 (Bar Chart) 由大到小排列本月各分類花費，直觀看出「錢都花在哪了」。
*   **日均花費**：自動計算本月已過天數的日均開銷，並推算「本月預估總支出」。

## 三、 資料結構設計 (邏輯層)

1.  **交易紀錄表 (Transactions)**
    *   `ID` (唯一識別碼，自動遞增)
    *   `Amount` (金額，數值)
    *   `Category_ID` (關聯分類表)
    *   `Note` (備註文字)
    *   `Tags` (標籤字串，以逗號分隔儲存)
    *   `Timestamp` (精確到秒的記帳時間，Epoch 整數)
    *   `Currency` (幣別代碼，如 TWD、RMB、USD，預設為本位幣)
2.  **分類表 (Categories)**
    *   `ID` (唯一識別碼)
    *   `Name` (分類名稱)
    *   `Icon_Code` (對應的 Material Icons 代碼)
    *   `Sort_Order` (排序權重)
    *   `Is_System` (是否為系統預設，防止誤刪)
3.  **預算表 (Budget)**
    *   `ID` (唯一識別碼)
    *   `Month` (預算月份，格式 YYYY-MM)
    *   `Amount` (預算金額，預設 15000)
4.  **設定表 (AppSettings)**
    *   `Key` (設定鍵名，如 `default_currency`、`budget_amount`)
    *   `Value` (設定值)

## 四、 UI/UX 設計原則

> 📐 **視覺設計參考**：`00-docs/figma/` 目錄下的 Figma 設計圖為 UI 實作的主要依據，包含以下頁面：
> - `首頁 - 今日概覽.png` — 首頁儀表板
> - `新增交易 - 輸入畫面.png` — 記帳輸入頁（含自訂數字鍵盤）
> - `明細列表 - 交易紀錄.png` — 時間軸列表
> - `統計報表 - 週-月統計.png` — 統計圖表頁
> - `設定 - 預算與帳戶.png` — 設定頁

1.  **分頁導航 (Bottom Navigation)**：
    *   Tab 1: **明細** (首頁，預設顯示今天，向下滾動看歷史)
    *   Tab 2: **統計** (圖表分析中心)
    *   Tab 3: **設定** (分類管理、預算、本位幣)
2.  **浮動操作按鈕 (FAB)**：
    *   在「明細」頁右下角放置一個顯眼的「+」號按鈕，隨時喚起記帳鍵盤。
3.  **視覺回饋**：
    *   記帳成功時，給予輕微的震動回饋 (Haptic Feedback) 與清脆的提示音，增加「完成感」。
    *   統計圖表支援「點擊交互」，例如點擊長條圖的某一天，下方列表自動篩選出該日的明細。
4.  **深色模式 (Dark Mode)**：跟隨系統設定，保護夜間查看時的視覺體驗。

## 五、 非功能性需求 (Non-functional Requirements)

1.  **離線可用性**：100% 離線運作，無網路狀態下所有功能（包含圖表生成）必須正常。
2.  **啟動速度**：冷啟動時間控制在 1 秒以內。
3.  **資料容量**：優化本地資料庫查詢，確保即使累積 5 年以上、數萬筆紀錄，列表滑動與圖表渲染依然流暢。
4.  **無廣告、無追蹤**：不請求不必要的手機權限（如位置、通訊錄），僅請求「FaceID/指紋」與「檔案存取（用於備份）」權限。

## 六、 未來擴展性 (Phase 2 預留)

雖然目前專注於「小額開銷」，但架構上可為未來的需求預留接口：
1.  **預算控制**：設定每月總預算或各分類預算，統計頁顯示「預算消耗進度條」，超支時顯示紅色警告。
2.  **定期定額項目**：如每月房租、串流媒體訂閱，設定後自動每月生成一筆紀錄，減少手動輸入。
3.  **資產帳戶切換**：從單純的「記帳」擴展為「記帳 + 記資產」，區分「從現金支付」或「從信用卡支付」，以便未來與你的 IBKR / HSBC 資產做對帳。
