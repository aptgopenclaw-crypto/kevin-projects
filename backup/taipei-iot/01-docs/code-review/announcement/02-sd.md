# Announcement 模組 SD 文件

## 1. 架構分層

```
AnnouncementController
    ├── AnnouncementService
    │       ├── AnnouncementRepository       → announcements
    │       ├── AnnouncementDeptRepository   → announcement_depts
    │       ├── AnnouncementReadRepository   → announcement_reads
    │       ├── DeptInfoRepository           → 查詢部門名稱
    │       └── UserRepository               → 查詢建立者顯示名稱
    └── AnnouncementReadService
            ├── AnnouncementRepository       → countUnread
            └── AnnouncementReadRepository   → markAsRead / markAllAsRead
```

---

## 2. Use Case Dataflow

---

### UC-1：管理者新增公告（POST /v1/auth/announcements）

```
[前端]
  填寫 title, content, status, scope, targetDeptIds, pinned, publishAt, expireAt
  → POST /v1/auth/announcements

[AnnouncementController.create()]
  → @PreAuthorize 驗證角色（ADMIN / SUPER_ADMIN / DEPT_ADMIN / ANNOUNCEMENT_MANAGE）
  → @AuditEvent(CREATE_ANNOUNCEMENT) 記錄操作審計

[AnnouncementService.create()]
  1. SecurityContextUtils.getUserInfo() 取得當前使用者資訊
  2. resolveDisplayName(userId) → UserRepository 查詢顯示名稱
  3. 判斷 DataScopeEnum：
     - 若 DataScopeEnum.ALL（ADMIN/SUPER_ADMIN）：
         scope 與 targetDeptIds 以 request 傳入值為主
     - 若非 ALL（DEPT_ADMIN）：
         強制 scope = DEPT
         強制 targetDeptIds = [user.getDeptId()]
  4. 建立 Announcement entity 並呼叫 announcementRepository.save()

  [INSERT INTO announcements]
    tenant_id     = TenantContext.getCurrentTenantId()（由 TenantEntityListener 自動填入）
    title         = request.title
    content       = request.content
    status        = request.status（DRAFT / PUBLISHED）
    scope         = DEPT_ADMIN 強制 DEPT，其餘依 request
    pinned        = request.pinned（null 則預設 false）
    publish_at    = request.publishAt（null 則使用 LocalDateTime.now()）
    expire_at     = request.expireAt
    created_by    = user.userId
    created_by_name = displayName
    created_at    = 自動產生
    updated_at    = 自動產生

  5. 若 scope = DEPT 且 targetDeptIds 不為空：
     → saveAnnouncementDepts(entity.id, targetDeptIds)

  [INSERT INTO announcement_depts]
    (announcement_id, dept_id) × targetDeptIds.size() 筆

  [announcement_reads]
    無操作（新公告無已讀紀錄）

  6. 回傳 AnnouncementResponse
```

**資料表異動摘要：**

| 表 | 操作 | 關鍵欄位 |
|----|------|---------|
| announcements | INSERT 1 筆 | id, tenant_id, title, scope, status, publish_at |
| announcement_depts | INSERT N 筆（scope=DEPT 才有） | announcement_id, dept_id |
| announcement_reads | 無異動 | - |

### UC-2：一般使用者查詢可見公告（GET /v1/auth/announcements）

```
[前端]
  → GET /v1/auth/announcements（不帶 admin=true）

[AnnouncementController.list()]
  → admin=false，呼叫 announcementService.listVisible(page, size)

[AnnouncementService.listVisible()]
  1. SecurityContextUtils.getUserInfo() 取得 userId, deptId
  2. deptId 為 null 則使用 -1L（排除部門公告）
  3. 呼叫 announcementRepository.findVisibleAnnouncements(deptId, now, pageable)

  [SELECT FROM announcements WHERE
    status = 'PUBLISHED'
    AND publish_at <= now
    AND (expire_at IS NULL OR expire_at > now)
    AND (scope = 'ALL'
         OR EXISTS (SELECT 1 FROM announcement_depts ad
                    WHERE ad.announcement_id = a.id AND ad.dept_id = :userDeptId))
    ORDER BY pinned DESC, publish_at DESC]

  4. toResponseList() 組裝回應資料：
     → 批次查詢 announcement_depts 取得各公告的 targetDeptIds
     → 批次查詢 announcement_reads 取得使用者已讀狀態
     → 組裝 isRead（是否已讀）、targetDeptNames（部門名稱）

  [SELECT FROM announcement_depts WHERE announcement_id IN (...)]
  [SELECT FROM announcement_reads WHERE announcement_id IN (...) AND user_id = :userId]

  5. 回傳 PageResponse<AnnouncementResponse>
```

**資料表異動摘要：**

| 表 | 操作 | 說明 |
|----|------|------|
| announcements | SELECT | 依 status / scope / publishAt / expireAt 篩選 |
| announcement_depts | SELECT | 取得各公告的部門關聯 |
| announcement_reads | SELECT | 判斷使用者已讀狀態 |

---

### UC-3：使用者標記單則公告已讀（POST /v1/auth/announcements/{id}/read）

```
[前端]
  使用者點選「已讀」或開啟公告詳情時觸發
  → POST /v1/auth/announcements/{id}/read

[AnnouncementController.markAsRead(id)]
  → announcementReadService.markAsRead(id)

[AnnouncementReadService.markAsRead()]
  1. SecurityContextUtils.getCurrentUserId() 取得 userId

  [INSERT INTO announcement_reads (announcement_id, user_id, read_at)
   VALUES (:announcementId, :userId, now())
   ON CONFLICT (announcement_id, user_id) DO NOTHING]
  
  → 若已有紀錄則不做任何變更（冪等設計）
  → 若無紀錄則新增一筆

  回傳：BaseResponse<Void>
```

**資料表異動摘要：**

| 表 | 操作 | 關鍵欄位 |
|----|------|---------|
| announcement_reads | INSERT（ON CONFLICT DO NOTHING） | announcement_id, user_id, read_at |

---

### UC-4：使用者全部標為已讀（POST /v1/auth/announcements/read-all）

```
[前端]
  使用者點選「全部標為已讀」
  → POST /v1/auth/announcements/read-all

[AnnouncementController.markAllAsRead()]
  → announcementReadService.markAllAsRead()

[AnnouncementReadService.markAllAsRead()]
  1. SecurityContextUtils.getUserInfo() 取得 userId, deptId
  2. TenantContext.getCurrentTenantId() 取得 tenantId
  3. deptId 為 null 則使用 -1L

  [批次 INSERT INTO announcement_reads (announcement_id, user_id, read_at)
   SELECT a.id, :userId, now()
   FROM announcements a
   WHERE a.tenant_id = :tenantId
     AND a.status = 'PUBLISHED'
     AND a.publish_at <= now()
     AND (a.expire_at IS NULL OR a.expire_at > now())
     AND (a.scope = 'ALL'
          OR EXISTS (SELECT 1 FROM announcement_depts ad
                     WHERE ad.announcement_id = a.id AND ad.dept_id = :userDeptId))
     AND NOT EXISTS (SELECT 1 FROM announcement_reads r
                     WHERE r.announcement_id = a.id AND r.user_id = :userId)
   ON CONFLICT (announcement_id, user_id) DO NOTHING]
  
  → 一次 SQL 完成所有符合條件的公告標記

  回傳：BaseResponse<Void>
```

**資料表異動摘要：**

| 表 | 操作 | 說明 |
|----|------|------|
| announcements | SELECT（子查詢） | 篩選可見且符合受眾的公告 |
| announcement_depts | EXISTS（子查詢） | 判斷部門受眾 |
| announcement_reads | 批次 INSERT（ON CONFLICT DO NOTHING） | 對所有未讀公告新增已讀紀錄 |

### UC-5：管理者編輯公告（PUT /v1/auth/announcements/{id}）

```
[前端]
  → PUT /v1/auth/announcements/{id}

[AnnouncementService.update()]
  1. 查詢 announcements 取得現有公告
  2. DEPT_ADMIN 驗證：只能更新 created_by = 自己的公告
  3. 更新欄位（title, content, status, scope, pinned, publishAt, expireAt）

  [UPDATE announcements SET
    title, content, status, scope, pinned, publish_at, expire_at, updated_at
    WHERE id = :id]

  4. 若 scope = DEPT：
     - DELETE FROM announcement_depts WHERE announcement_id = :id
     - INSERT INTO announcement_depts (announcement_id, dept_id) × newDeptIds
  5. 若 scope = ALL：
     - DELETE FROM announcement_depts WHERE announcement_id = :id（清除舊部門關聯）
```

**資料表異動摘要：**

| 表 | 操作 | 關鍵欄位 |
|----|------|---------|
| announcements | UPDATE | title, content, status, scope, pinned, publish_at, expire_at |
| announcement_depts | DELETE + INSERT | 先清後補，對應新 targetDeptIds |
| announcement_reads | 無異動 | - |

---

### UC-6：管理者刪除公告（DELETE /v1/auth/announcements/{id}）

```
[前端]
  → DELETE /v1/auth/announcements/{id}

[AnnouncementService.delete()]
  1. DELETE FROM announcement_depts WHERE announcement_id = :id
  2. DELETE FROM announcements WHERE id = :id
```

**資料表異動摘要：**

| 表 | 操作 | 說明 |
|----|------|------|
| announcement_depts | DELETE | 清除部門關聯 |
| announcements | DELETE | 刪除公告主資料 |
| announcement_reads | 無異動（歷史已讀紀錄保留） | - |

## 3. 多租戶設計

- `announcements` 具有 `tenant_id` 欄位，並配置 Hibernate `@Filter(name = "tenantFilter")`。
- `TenantEntityListener` 於新增時自動填入 `TenantContext.getCurrentTenantId()`。
- 所有查詢自動套用 tenant 過濾，不同租戶的公告完全隔離。

---

## 4. 冪等設計

- `markAsRead`：使用 `ON CONFLICT (announcement_id, user_id) DO NOTHING`，重複呼叫不會出錯。
- `markAllAsRead`：同上，確保批次標記操作的安全性。

---

## 5. 資料表關聯圖

```
announcements (1)
    ├─── announcement_depts (N)   scope=DEPT 才有紀錄
    └─── announcement_reads (N)   使用者標記已讀後產生

announcement_depts:
    FK: announcement_id → announcements.id
    FK: dept_id → (部門表，無外鍵約束，邏輯關聯)

announcement_reads:
    UNIQUE (announcement_id, user_id)
    FK: announcement_id → announcements.id（邏輯）
    FK: user_id → users.user_id（邏輯）
```
