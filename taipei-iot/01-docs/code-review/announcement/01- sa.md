# Announcement 模組 SA 文件

## 1. 模組簡介

公告模組提供系統公告的新增、編輯、刪除、查詢、已讀追蹤與未讀計數功能。  
支援多租戶（Tenant）隔離、多角色權限（ADMIN / DEPT_ADMIN / 一般使用者）、公告受眾範圍（全體 ALL / 指定部門 DEPT）。

---

## 2. 資料表結構

### 2.1 `announcements`（公告主表）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | bigserial PK | 公告 ID |
| tenant_id | varchar(50) | 租戶 ID（多租戶隔離） |
| title | varchar(200) | 公告標題 |
| content | text | 公告內容 |
| status | varchar(20) | 狀態：DRAFT / PUBLISHED |
| scope | varchar(20) | 受眾：ALL / DEPT |
| pinned | boolean | 是否置頂 |
| publish_at | timestamptz | 發佈時間（null 表示即時） |
| expire_at | timestamptz | 到期時間（null 表示永不到期） |
| created_by | varchar(50) | 建立者 user_id |
| created_by_name | varchar(100) | 建立者顯示名稱 |
| created_at | timestamptz | 建立時間（自動） |
| updated_at | timestamptz | 更新時間（自動） |

### 2.2 `announcement_depts`（公告部門關聯表）

| 欄位 | 型別 | 說明 |
|------|------|------|
| announcement_id | bigint PK (FK) | 公告 ID |
| dept_id | bigint PK | 部門 ID |

> 複合主鍵 (announcement_id, dept_id)  
> 僅當 scope = DEPT 時，此表才會有對應紀錄

### 2.3 `announcement_reads`（公告已讀紀錄表）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | bigserial PK | 紀錄 ID |
| announcement_id | bigint | 公告 ID |
| user_id | varchar(50) | 使用者 ID |
| read_at | timestamptz | 標記已讀時間 |

> 具備 UNIQUE 約束 (announcement_id, user_id)，防止重複已讀紀錄

## 3. API 清單

### 3.1 GET /v1/auth/announcements — 查詢公告列表

**目的：** 取得分頁公告列表。一般使用者取得前台可見公告；管理員帶 `admin=true` 取得後台管理用全部公告。

**Request 參數：**

| 參數 | 預設值 | 說明 |
|------|--------|------|
| admin | false | true = 管理員模式（需具備管理權限） |
| statusFilter | ALL | DRAFT / PUBLISHED / EXPIRED / ALL |
| keyword | - | 標題關鍵字模糊搜尋 |
| page | 0 | 頁碼（0-based） |
| size | 10 | 每頁筆數 |

**資料表操作：** `announcements`（SELECT）、`announcement_depts`（子查詢）、`announcement_reads`（子查詢）

**查詢邏輯差異：**

| 角色 | 查詢範圍 |
|------|----------|
| 一般使用者 | status=PUBLISHED + publishAt<=now + 未過期 + scope=ALL 或 dept 符合 |
| ADMIN / SUPER_ADMIN | 所有公告（依 statusFilter / keyword 篩選） |
| DEPT_ADMIN | 自己建立的 + scope=DEPT 且受眾含自己部門的 |

**排序：** pinned DESC, publish_at DESC

---

### 3.2 GET /v1/auth/announcements/{id} — 查詢單筆公告

**目的：** 取得指定 id 的公告詳情，含已讀狀態與 editable 標記。

**資料表操作：** `announcements`（SELECT by id）

---

### 3.3 GET /v1/auth/announcements/unread-count — 取得未讀公告數

**目的：** 取得目前登入使用者的未讀公告數量，用於前端通知 badge 顯示。

**資料表操作：**
- `announcements`（COUNT，條件：status=PUBLISHED + 未過期 + 受眾符合）
- `announcement_reads`（NOT EXISTS 子查詢，確認未讀）

**回傳：** `UnreadCountResponse { count: int }`

---

### 3.4 POST /v1/auth/announcements/{id}/read — 標記單則已讀

**目的：** 將指定公告標記為當前使用者已讀。

**資料表操作：**
- `announcement_reads`（INSERT ON CONFLICT DO NOTHING）

**關鍵欄位：** announcement_id、user_id（取自 SecurityContext）、read_at（now()）

---

### 3.5 POST /v1/auth/announcements/read-all — 全部標為已讀

**目的：** 將當前使用者所有可見且未讀的公告一次全部標記為已讀。

**資料表操作：**
- `announcement_reads`（批次 INSERT，子查詢取得符合受眾且未讀的公告）
- `announcements`（SELECT 子查詢，用於篩選可見公告）
- `announcement_depts`（EXISTS 子查詢）

**條件：** status=PUBLISHED + publishAt<=now + 未過期 + scope=ALL 或 dept 符合 + 未讀

---

### 3.6 POST /v1/auth/announcements — 新增公告

**目的：** 管理者建立一則新公告，可指定受眾為全體或特定部門。

**需要權限：** ADMIN / SUPER_ADMIN / DEPT_ADMIN 或 ANNOUNCEMENT_MANAGE

**Request Body（AnnouncementRequest）：**

| 欄位 | 必填 | 說明 |
|------|------|------|
| title | Y | 標題，max 200 |
| content | Y | 內容（文字） |
| status | Y | DRAFT / PUBLISHED |
| scope | Y | ALL / DEPT |
| targetDeptIds | 條件 | scope=DEPT 時必填，DEPT_ADMIN 強制鎖自己部門 |
| pinned | N | 是否置頂，預設 false |
| publishAt | N | 排程發佈時間，null 表示使用建立時間 |
| expireAt | N | 到期時間 |

**資料表操作：**
- `announcements`（INSERT）：寫入標題、內容、狀態、scope、pinned、publishAt、expireAt、created_by、created_by_name
- `announcement_depts`（INSERT）：scope=DEPT 時，對每個 targetDeptId 寫入一筆 (announcement_id, dept_id)

**審核記錄：** AuditEventType.CREATE_ANNOUNCEMENT

---

### 3.7 PUT /v1/auth/announcements/{id} — 編輯公告

**目的：** 管理者更新公告內容或受眾設定。

**需要權限：** ADMIN / SUPER_ADMIN / DEPT_ADMIN 或 ANNOUNCEMENT_MANAGE

**資料表操作：**
- `announcements`（UPDATE）：更新 title、content、status、scope、pinned、publishAt、expireAt
- `announcement_depts`（DELETE + INSERT）：先刪除舊的關聯紀錄，再重新寫入新的 targetDeptIds

**DEPT_ADMIN 限制：** 只能編輯自己建立的公告。

**審核記錄：** AuditEventType.UPDATE_ANNOUNCEMENT

---

### 3.8 DELETE /v1/auth/announcements/{id} — 刪除公告

**目的：** 管理者刪除指定公告。

**需要權限：** ADMIN / SUPER_ADMIN / DEPT_ADMIN 或 ANNOUNCEMENT_MANAGE

**資料表操作：**
- `announcements`（DELETE）
- `announcement_depts`（DELETE by announcement_id，cascade 清除部門關聯）

**審核記錄：** AuditEventType.DELETE_ANNOUNCEMENT

## 4. 權限矩陣

| 功能 | 一般使用者 | DEPT_ADMIN | ADMIN / SUPER_ADMIN | ANNOUNCEMENT_MANAGE |
|------|-----------|------------|---------------------|---------------------|
| 查詢可見公告 | ✓ | ✓ | ✓ | ✓ |
| 查詢管理列表 | ✗ | ✓（限自己相關） | ✓（全部） | ✓ |
| 未讀數查詢 | ✓ | ✓ | ✓ | ✓ |
| 標記已讀 | ✓ | ✓ | ✓ | ✓ |
| 新增公告 | ✗ | ✓（強制 scope=DEPT） | ✓ | ✓ |
| 編輯公告 | ✗ | ✓（限自己建立） | ✓ | ✓ |
| 刪除公告 | ✗ | ✓ | ✓ | ✓ |

## 5. Enum 定義

### AnnouncementStatus
| 值 | 說明 |
|----|------|
| DRAFT | 草稿，不對前台顯示 |
| PUBLISHED | 已發佈，依 publishAt / expireAt 決定是否顯示 |

### AnnouncementScope
| 值 | 說明 |
|----|------|
| ALL | 全體租戶內使用者可見 |
| DEPT | 僅 targetDeptIds 指定部門的使用者可見 |
